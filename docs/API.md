# API Reference

Complete API documentation for Market API endpoints.

## Base URL

- **Local**: `http://localhost:5000`
- **Docker**: `http://localhost:5000`
- **HTTPS**: `https://localhost:7294` (Development)

## Authentication

Currently, the API does not require authentication. This can be added in future versions.

## Endpoints

### Products

#### Get All Products

Retrieves all products from the database.

**Endpoint**: `GET /api/products`

**Response**: `200 OK`

```json
[
  {
    "id": "507f1f77bcf86cd799439011",
    "name": "Laptop",
    "price": 999.99
  },
  {
    "id": "507f1f77bcf86cd799439012",
    "name": "Mouse",
    "price": 29.99
  }
]
```

**Example**:
```bash
curl http://localhost:5000/api/products
```

---

#### Get Product by ID

Retrieves a specific product by its ID.

**Endpoint**: `GET /api/products/{id}`

**Parameters**:
- `id` (string, required): MongoDB ObjectId of the product

**Response**: `200 OK`

```json
{
  "id": "507f1f77bcf86cd799439011",
  "name": "Laptop",
  "price": 999.99
}
```

**Example**:
```bash
curl http://localhost:5000/api/products/507f1f77bcf86cd799439011
```

**Error Responses**:
- `404 Not Found`: Product not found

---

#### Get Products by Price Range

Retrieves products within a specified price range.

**Endpoint**: `GET /api/products/GetByPriceRange/{minPrice}/{maxPrice}`

**Parameters**:
- `minPrice` (decimal, required): Minimum price
- `maxPrice` (decimal, required): Maximum price

**Response**: `200 OK`

```json
[
  {
    "id": "507f1f77bcf86cd799439011",
    "name": "Laptop",
    "price": 999.99
  },
  {
    "id": "507f1f77bcf86cd799439013",
    "name": "Keyboard",
    "price": 79.99
  }
]
```

**Example**:
```bash
curl http://localhost:5000/api/products/GetByPriceRange/50/1000
```

---

#### Create Product

Creates a new product.

**Endpoint**: `POST /api/products`

**Request Headers**:
```
Content-Type: application/json
```

**Request Body**:
```json
{
  "name": "Laptop",
  "price": 999.99
}
```

**Response**: `201 Created`

```json
{
  "id": "507f1f77bcf86cd799439011",
  "name": "Laptop",
  "price": 999.99
}
```

**Response Headers**:
```
Location: /api/products/507f1f77bcf86cd799439011
```

**Example**:
```bash
curl -X POST http://localhost:5000/api/products \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Laptop",
    "price": 999.99
  }'
```

**Validation Rules**:
- `name`: Required, string
- `price`: Required, decimal, must be positive

**Error Responses**:
- `400 Bad Request`: Invalid request body

---

#### Update Product

Updates an existing product.

**Endpoint**: `PUT /api/products/{id}`

**Parameters**:
- `id` (string, required): MongoDB ObjectId of the product

**Request Headers**:
```
Content-Type: application/json
```

**Request Body**:
```json
{
  "name": "Gaming Laptop",
  "price": 1299.99
}
```

**Response**: `200 OK`

```json
{
  "id": "507f1f77bcf86cd799439011",
  "name": "Gaming Laptop",
  "price": 1299.99
}
```

**Example**:
```bash
curl -X PUT http://localhost:5000/api/products/507f1f77bcf86cd799439011 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Gaming Laptop",
    "price": 1299.99
  }'
```

**Error Responses**:
- `404 Not Found`: Product not found
- `400 Bad Request`: Invalid request body

---

#### Delete Product

Deletes a product by ID.

**Endpoint**: `DELETE /api/products/{id}`

**Parameters**:
- `id` (string, required): MongoDB ObjectId of the product

**Response**: `200 OK`

```json
{
  "success": true,
  "message": "Product deleted"
}
```

**Example**:
```bash
curl -X DELETE http://localhost:5000/api/products/507f1f77bcf86cd799439011
```

**Error Responses**:
- `404 Not Found`: Product not found

## Data Models

### Product

```json
{
  "id": "string (MongoDB ObjectId)",
  "name": "string",
  "price": "decimal"
}
```

**Field Descriptions**:
- `id`: Unique identifier (auto-generated by MongoDB)
- `name`: Product name
- `price`: Product price in decimal format

---

## Error Handling

All endpoints return standard HTTP status codes:

- `200 OK`: Request succeeded
- `201 Created`: Resource created successfully
- `400 Bad Request`: Invalid request data
- `404 Not Found`: Resource not found
- `500 Internal Server Error`: Server error

**Error Response Format**:
```json
{
  "type": "https://tools.ietf.org/html/rfc7231#section-6.5.1",
  "title": "One or more validation errors occurred.",
  "status": 400,
  "errors": {
    "Name": ["The Name field is required."]
  }
}
```
